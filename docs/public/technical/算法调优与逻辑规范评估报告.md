#### 方案二：增量指标更新

**问题**：每日新增一行数据，却重新计算全部历史指标

**解决方案**：

```python
from collections import deque

class IncrementalSMA:
    """增量SMA计算"""
    
    def __init__(self, window: int):
        self.window = window
        self.history = deque(maxlen=window)
        self.current_sum = 0.0
    
    def update(self, new_price: float) -> float:
        """更新并返回新SMA"""
        if len(self.history) == self.window:
            self.current_sum -= self.history[0]
        
        self.history.append(new_price)
        self.current_sum += new_price
        
        return self.current_sum / len(self.history)

class IncrementalEMA:
    """增量EMA计算"""
    
    def __init__(self, window: int):
        self.multiplier = 2 / (window + 1)
        self.last_ema = None
    
    def update(self, new_price: float) -> float:
        """更新并返回新EMA"""
        if self.last_ema is None:
            self.last_ema = new_price
        else:
            self.last_ema = (new_price - self.last_ema) * self.multiplier + self.last_ema
        
        return self.last_ema
```

**预期收益**：
- 增量更新复杂度：O(N) → O(1)
- 每日向量库更新速度：**提升10倍以上**

### 1.3 向量检索优化（P0）

#### 方案一：切换至FAISS（强烈推荐）

**问题**：ChromaDB基于SQLite+HNSW，单机千万级向量时检索延迟50-200ms

**对比分析**：

| 对比项 | ChromaDB | FAISS | 提升 |
|--------|----------|-------|------|
| 检索延迟（百万级） | 50-200ms | **<5ms** | **10-40倍** |
| 索引构建时间 | 较慢 | 极快（GPU加速） | 5-10倍 |
| 增量更新 | 支持 | 需重建或使用IVF | 持平 |
| 内存占用 | 较高 | 极低（支持内存映射） | 50-70% |
| 分布式支持 | 不支持 | 支持 | N/A |

**实现方案**：

```python
import faiss
import numpy as np
from typing import List, Dict, Tuple

class FAISSAdapter:
    """FAISS向量数据库适配器"""
    
    def __init__(self, dim: int, metric: str = 'ip'):
        """
        初始化FAISS索引
        
        Args:
            dim: 向量维度
            metric: 距离度量（'ip'=内积，'l2'=欧氏距离）
        """
        if metric == 'ip':
            # 内积（等价于余弦相似度，需归一化向量）
            self.index = faiss.IndexFlatIP(dim)
        elif metric == 'l2':
            # 欧氏距离
            self.index = faiss.IndexFlatL2(dim)
        else:
            raise ValueError(f"Unsupported metric: {metric}")
        
        self.metadata_list: List[Dict] = []
    
    def insert(self, vectors: np.ndarray, metadata: List[Dict]):
        """插入向量"""
        # 归一化（用于内积计算余弦相似度）
        faiss.normalize_L2(vectors)
        
        self.index.add(vectors)
        self.metadata_list.extend(metadata)
    
    def search(self, query: np.ndarray, k: int = 10) -> Tuple[np.ndarray, np.ndarray, List[Dict]]:
        """
        检索相似向量
        
        Returns:
            distances: 距离数组
            indices: 索引数组
            metadata: 元数据列表
        """
        faiss.normalize_L2(query)
        distances, indices = self.index.search(query, k)
        
        results = [self.metadata_list[i] for i in indices[0] if i != -1]
        return distances[0], indices[0], results
    
    def save(self, path: str):
        """保存索引"""
        faiss.write_index(self.index, path)
        # 保存元数据
        import json
        with open(f"{path}.meta", 'w') as f:
            json.dump(self.metadata_list, f)
    
    def load(self, path: str):
        """加载索引"""
        self.index = faiss.read_index(path)
        # 加载元数据
        import json
        with open(f"{path}.meta", 'r') as f:
            self.metadata_list = json.load(f)
```

**预期收益**：
- 检索延迟：**50-200ms → <10ms**
- 支持百万级向量实时检索

```python
def compute_weights_improved(
    distances: np.ndarray,
    method: str = 'inverse_linear',
    max_weight_ratio: float = 3.0,
    temporal_decay: float = 0.05
) -> np.ndarray:
    """
    改进的权重计算
    
    Args:
        distances: 距离数组
        method: 权重方法
        max_weight_ratio: 最大权重/最小权重比例上限
        temporal_decay: 时间衰减系数
    
    Returns:
        归一化权重数组
    """
    if method == 'inverse_linear':
        # 方案1: 1/(1+d) 避免除零
        weights = 1.0 / (1.0 + distances)
    
    elif method == 'exponential':
        # 方案2: 指数衰减
        weights = np.exp(-distances * 10)
    
    elif method == 'gaussian':
        # 方案3: 高斯核
        sigma = 0.01
        weights = np.exp(-distances**2 / (2 * sigma**2))
    
    else:
        raise ValueError(f"Unknown method: {method}")
    
    # 应用权重上限约束
    min_weight = weights.min()
    max_allowed = min_weight * max_weight_ratio
    weights = np.clip(weights, min_weight, max_allowed)
    
    # 应用时间衰减（可选）
    if temporal_decay > 0:
        # 根据样本日期应用衰减
        # 实现略
        pass
    
    # 归一化
    return weights / weights.sum()
```

**改进效果**：
- 最大权重限制为最小权重的3倍
- 避免单样本绑架
- 提升预测平滑度

```python
def synthesize_prediction(similar_states):

    returns = [s.future_return for s in similar_states]
```

#### 改进方案

```python
def synthesize_prediction_improved(similar_states, weights):
    """
    改进的预测合成
    
    Args:
        similar_states: 相似状态列表
        weights: 对应权重
    
    Returns:
        预测结果字典
    """
    returns = np.array([s.future_return for s in similar_states])
    
    # 1. 加权平均收益
    weighted_avg_return = np.average(returns, weights=weights)
    
    # 2. 加权上涨概率
    up_mask = (returns > 0).astype(float)
    weighted_up_prob = np.average(up_mask, weights=weights)
    
    # 3. 计算波动率（加权标准差）
    weighted_std = np.sqrt(np.average((returns - weighted_avg_return)**2, weights=weights))
    
    # 4. 计算下行风险
    down_returns = returns[returns < 0]
    down_risk = np.sqrt(np.mean(down_returns**2)) if len(down_returns) > 0 else 0.0
    
    # 5. 标签分布熵值（不确定性度量）
    labels = [s.label for s in similar_states]
    label_entropy = calculate_entropy(labels)
    
    # 6. 综合置信度
    confidence = compute_confidence(
        up_prob=weighted_up_prob,
        sample_quality=len(similar_states) / 20.0,
        similarity_strength=1.0 / (1.0 + np.mean(weights)),
        uncertainty=1.0 - label_entropy
    )
    
    return {
        'avg_return': weighted_avg_return,
        'up_probability': weighted_up_prob,
        'volatility': weighted_std,
        'down_risk': down_risk,
        'label_entropy': label_entropy,
        'confidence': confidence,
        'sample_size': len(similar_states)
    }

def compute_confidence(up_prob, sample_quality, similarity_strength, uncertainty):
    """多维度置信度评估"""
    weights = {
        'up_prob': 0.40,
        'sample_quality': 0.25,
        'similarity_strength': 0.20,
        'uncertainty': 0.15
    }
    
    confidence = (
        weights['up_prob'] * up_prob +
        weights['sample_quality'] * min(sample_quality, 1.0) +
        weights['similarity_strength'] * similarity_strength +
        weights['uncertainty'] * uncertainty
    )
    
    return min(max(confidence, 0.0), 1.0)
```

**改进效果**：
- 预测输出更丰富（波动率、下行风险、熵值）
- 置信度评估更科学
- 支持风险管理和仓位计算

## 六、总结与建议

### 6.1 核心结论

1. **计算效率问题非架构性缺陷**
   - 当前延迟主要来自工程实现未优化
   - 通过缓存+FAISS+离线预计算可解决95%延迟

2. **权重算法存在设计缺陷**
   - `inverse`方法导致极近样本权重爆炸
   - 必须立即修正为`inverse_linear`+上限约束

3. **算法规范性需提升**
   - 类型注解、Docstring、测试覆盖率不足
   - 预测合成逻辑过于简单

4. **优化投入产出比极高**
   - 1-2周可实现95%性能提升
   - 代码质量改善提升可维护性

### 6.2 实施建议

**第一阶段（紧急，1周内）**：
1. 权重算法修正
2. 特征计算缓存
3. 查询结果LRU缓存

**第二阶段（重要，2-3周）**：
1. 切换FAISS
2. 分离离线/在线计算
3. 特征重要性分析

**第三阶段（优化，1-2月）**：
1. 增量指标更新
2. 向量量化压缩
3. 完善测试覆盖

### 6.3 风险提示

⚠️ **切换FAISS需注意**：
- 增量更新需重建索引或使用IVF
- 元数据存储需额外实现
- 建议通过适配器模式平滑迁移

⚠️ **离线预计算需注意**：
- 需保证离线和在线特征一致性
- Scaler和PCA模型版本管理
- 定时任务失败重试机制
