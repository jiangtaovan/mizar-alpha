## 🔧 开发环境搭建

### 1. 克隆仓库

```bash
git clone https://github.com/jiangtaovan/mizar-alpha.git
cd mizar-alpha
```

### 2. 创建虚拟环境

```bash
# 使用 venv
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate     # Windows

# 或使用 conda
conda create -n mizar python=3.10
conda activate mizar
```

### 3. 安装依赖

```bash
# 安装开发依赖
pip install -e ".[dev]"

# 或手动安装
pip install -r requirements-dev.txt
```

### 4. 运行测试

```bash
# 运行所有测试
pytest tests/ -v

# 运行测试并生成覆盖率报告
pytest tests/ --cov=mizar --cov-report=html

# 查看覆盖率
open htmlcov/index.html  # Mac/Linux
start htmlcov\index.html  # Windows
```

## 🔄 Git 工作流

### 分支策略

```
main (稳定版本)
  ↑
  └── develop (开发分支)
        ↑
        ├── feature/xxx (新功能)
        ├── bugfix/xxx (Bug修复)
        └── docs/xxx (文档更新)
```

### 提交规范

#### Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

#### Type 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| feat | 新功能 | `feat: 添加多周期信号融合` |
| fix | Bug修复 | `fix: 修复时间对齐问题` |
| docs | 文档 | `docs: 更新API使用指南` |
| style | 代码格式 | `style: 格式化预测服务代码` |
| refactor | 重构 | `refactor: 优化向量检索逻辑` |
| perf | 性能 | `perf: 提升批量预测速度` |
| test | 测试 | `test: 添加回测引擎测试` |
| chore | 构建/工具 | `chore: 更新依赖版本` |

#### 示例

```bash
# 好的提交
git commit -m "feat: 添加动态置信度调整机制

根据市场状态自动调整置信度阈值，提升策略适应性。

- 实现牛市/熊市/震荡市识别
- 配置化阈值调整参数
- 添加单元测试

Closes #123"

# 不好的提交
git commit -m "更新代码"
```

### Pull Request 流程

1. **Fork 仓库**
2. **创建分支**: `git checkout -b feature/my-feature`
3. **开发并提交**
4. **推送到远程**: `git push origin feature/my-feature`
5. **创建 PR**
6. **通过 CI 检查**
7. **代码审查**
8. **合并到 develop**

## 🧪 测试要求

### 单元测试

```python
# tests/test_prediction_service.py
import pytest
from mizar import PredictionService

class TestPredictionService:
    """预测服务测试"""
    
    @pytest.fixture
    def service(self):
        """测试夹具"""
        return PredictionService(test_mode=True)
    
    def test_predict_success(self, service):
        """测试预测成功"""
        result = service.predict('000001', '2026-04-21')
        
        assert result.up_probability >= 0
        assert result.up_probability <= 1
        assert len(result.similar_states) > 0
    
    def test_predict_invalid_symbol(self, service):
        """测试无效股票代码"""
        with pytest.raises(ValueError):
            service.predict('INVALID', '2026-04-21')
```

| 模块类型 | 最低覆盖率 |

## 🚀 发布流程

### 版本号管理

遵循语义化版本：`MAJOR.MINOR.PATCH`

- **MAJOR**: 不兼容的 API 变更
- **MINOR**: 向后兼容的功能新增
- **PATCH**: 向后兼容的问题修复

### 发布步骤

1. **更新版本号**
   ```python
   # mizar/__init__.py
   __version__ = "1.1.0"
   ```

2. **更新 CHANGELOG**
   ```markdown
   ## [1.1.0] - 2026-04-21
   
   ### 新增
   - 添加XXX功能
   
   ### 修复
   - 修复XXX问题
   ```

3. **创建发布分支**
   ```bash
   git checkout -b release/1.1.0
   ```

4. **最终测试**
   ```bash
   pytest tests/ -v
   ```

5. **合并到 main**
   ```bash
   git checkout main
   git merge release/1.1.0
   ```

6. **打标签**
   ```bash
   git tag -a v1.1.0 -m "Release v1.1.0"
   git push origin main --tags
   ```

7. **发布到 PyPI**
   ```bash
   python -m build
   python -m twine upload dist/*
   ```

## 💡 常见问题

### Q: 如何选择合适的 Issue？

A: 查看 GitHub Issues，标签说明：
- `good first issue`: 适合新手
- `help wanted`: 急需帮助
- `bug`: Bug 修复
- `enhancement`: 功能增强

### Q: 开发周期多长？

A: 
- 简单 Bug 修复: 1-3 天
- 小功能: 1-2 周
- 大功能: 2-4 周
