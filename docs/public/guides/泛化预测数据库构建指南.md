# Mizarα 泛化预测数据库构建指南

> 跨市场、多周期预测数据库的设计原则与最佳实践
> 
> **适用场景**: A股、港股、美股、加密货币、期货等  
> **数据周期**: 日线、分钟线、周线等  
> **版本**: v1.0

---

## 📋 概述

本指南基于 Mizarα 在 A 股日线数据上的实证经验（45个单点预测样本、601766回测2.84年数据），总结构建**泛化预测数据库**的核心原则和注意事项，帮助用户快速适配新市场和新周期。

### 为什么需要泛化设计？

根据我们的实证评估发现：

| 发现 | 影响 | 启示 |
|------|------|------|
| 趋势市准确率77.8%，反转市0% | 场景依赖性强 | 需多市场数据覆盖不同行情 |
| 跨标的检索提升样本量5-10倍 | 单股样本稀疏 | 需多样化标的增强泛化 |
| 中性区间（40-60%）准确率50% | 信号质量不稳定 | 需大规模数据训练置信度评估 |
| 权重算法导致单样本绑架 | 小样本风险高 | 需足够样本量和多样性 |

**核心结论**：当前基于A股日线的架构在**低波动蓝筹股、趋势行情**中表现良好，但要对其他场景（加密货币、分钟K线、高波动市场）保持有效，必须在数据层面进行系统性设计。

### 1.1 时间区间选择

#### 基本原则

| 市场类型 | 最小区间 | 推荐区间 | 原因 |
|---------|---------|---------|------|
| **成熟股市**（A股、美股） | 3年 | 5-10年 | 覆盖牛熊周期 |
| **新兴市场** | 2年 | 3-5年 | 市场结构变化快 |
| **加密货币** | 1年 | 2-3年 | 市场演进快速，旧数据参考价值低 |
| **期货市场** | 3年 | 5-8年 | 合约周期性、季节性特征 |

#### 关键周期覆盖

**必须包含的市场阶段**：

```
✅ 牛市阶段（趋势上涨）
   - 特征：均线多头排列、成交量放大
   - 作用：学习趋势跟踪模式
   - 示例：A股2024-2025年

✅ 熊市阶段（趋势下跌）
   - 特征：均线空头排列、恐慌性抛售
   - 作用：学习做空信号和风险控制
   - 示例：A股2022年

✅ 震荡市（横盘整理）
   - 特征：窄幅波动、无明确方向
   - 作用：学习过滤无效信号
   - 示例：A股2023年上半年

✅ 极端行情（暴涨暴跌）
   - 特征：涨跌停、流动性枯竭
   - 作用：学习风险识别
   - 示例：2020年疫情初期、2021年核心资产泡沫破裂

✅ 结构性行情（板块轮动）
   - 特征：指数平稳、板块分化
   - 作用：学习行业特性
   - 示例：2024年AI板块独立行情
```

### 1.3 避免过拟合风险

#### 过拟合风险来源

根据实证评估发现的风险：

| 风险类型 | 表现 | 案例 |
|---------|------|------|
| **单样本绑架** | 极近样本（距离<0.005）权重过高 | 003036偏移3误判 |
| **样本集中** | 检索到的样本过度集中在少数股票 | 300685偏移6误判 |
| **时间集中** | 样本集中在特定时间段 | 缺乏跨周期验证 |
| **特征冗余** | 52个特征中高度相关 | 增加计算开销，降低泛化 |
| **参数过调** | Top-K、权重参数过度优化 | 样本内好，样本外差 |

#### 防范策略

**1. 样本量要求**

```
最小样本量计算：
- 每个场景（牛/熊/震荡）至少 500 个样本
- 每个行业至少 100 个样本
- 总体本量 ≥ 10,000 个状态向量

当前A股日线配置：
- 100只股票 × 500交易日 = 50,000 个样本 ✅ 充足
- 但需注意行业分布均衡性
```

```python
from sklearn.model_selection import TimeSeriesSplit

def cross_validate_strategy(data, n_splits=5):
    """时间序列交叉验证"""
    tscv = TimeSeriesSplit(n_splits=n_splits)
    
    results = []
    for train_idx, test_idx in tscv.split(data):
        train_data = data.iloc[train_idx]
        test_data = data.iloc[test_idx]
        
        # 在训练集上优化参数
        params = optimize_params(train_data)
        
        # 在测试集上验证
        result = validate_params(params, test_data)
        results.append(result)
    
    # 检查稳定性
    check_stability(results)
    
    return results

def check_stability(results):
    """检查参数稳定性"""
    returns = [r['total_return'] for r in results]
    sharpe = [r['sharpe_ratio'] for r in results]
    
    # 收益标准差应小于均值的30%
    assert np.std(returns) / np.mean(returns) < 0.30, "收益不稳定"
    
    # 夏普比率应全部为正
    assert all(s > 0 for s in sharpe), "部分周期夏普为负"
    
    # 最大回撤应在合理范围
    assert all(abs(r['max_drawdown']) < 0.20 for r in results), "回撤过大"
```

```python
def remove_redundant_features(feature_matrix, threshold=0.95):
    """移除高度相关特征"""
    corr_matrix = np.corrcoef(feature_matrix.T)
    
    to_remove = []
    for i in range(len(corr_matrix)):
        for j in range(i+1, len(corr_matrix)):
            if abs(corr_matrix[i, j]) > threshold:
                # 保留方差更大的特征
                if np.var(feature_matrix[:, i]) < np.var(feature_matrix[:, j]):
                    to_remove.append(i)
                else:
                    to_remove.append(j)
    
    return list(set(to_remove))
```

```python
def parameter_robustness_test():
    """参数鲁棒性测试"""
    param_grid = {
        'top_k': [5, 10, 15, 20],
        'distance_metric': ['euclidean', 'cosine'],
        'weight_method': ['inverse_linear', 'exponential'],
        'max_weight_ratio': [2.0, 3.0, 5.0],
    }
    
    results = []
    for params in parameter_combinations(param_grid):
        result = run_backtest(params)
        results.append({**params, **result})
    
    # 检查最优参数是否显著优于其他参数
    best = max(results, key=lambda x: x['sharpe_ratio'])
    others = [r for r in results if r != best]
    
    # 最优参数应至少比中位数好20%
    median_sharpe = np.median([r['sharpe_ratio'] for r in others])
    assert best['sharpe_ratio'] > median_sharpe * 1.20, "参数过拟合"
    
    # 相邻参数表现应平滑
    check_parameter_smoothness(results)
    
    return best
```

```python
    def __init__(self, market_type='cn_stock', timeframe='1d'):
        # 根据场景加载配置
        self.config = load_config(market_type, timeframe)
        
        # 动态特征工程
        self.feature_engineer = FeatureEngineer(self.config)

        # 动态时间衰减
        self.temporal_decay = self.config.temporal_decay
    
    def predict(self, symbol, timestamp):
        """支持多市场、多周期预测"""
        # 1. 计算特征（缓存优化）
        features = self.feature_engineer.compute_cached(symbol, timestamp)
        
        # 2. 归一化+PCA
        normalized = self.feature_engineer.normalize(features)
        reduced = self.feature_engineer.reduce(normalized)
        
        # 3. 检索（FAISS优化）
        similar_states = self.db.search(reduced, k=self.config.top_k)
        
        # 4. 预测合成（改进权重）
        prediction = synthesize_prediction_improved(
            similar_states,
            weights_method=self.config.weight_method
        )
        
        return prediction
```

### 3.3 快速验证框架

```python
class QuickValidationFramework:
    """快速验证框架"""
    
    def __init__(self, market_type, timeframe):
        self.market_type = market_type
        self.timeframe = timeframe
        self.config = load_config(market_type, timeframe)
    
    def run_validation(self, data, test_ratio=0.2):
        """运行完整验证流程"""
        # 1. 数据检查
        self.check_data_quality(data)
        
        # 2. 特征工程
        features = self.build_features(data)
        
        # 3. 构建向量库
        vector_db = self.build_vector_db(features)
        
        # 4. 样本内测试
        in_sample = self.test_in_sample(vector_db)
        
        # 5. 样本外测试
        out_sample = self.test_out_sample(vector_db, test_ratio)
        
        # 6. 生成报告
        report = self.generate_report(in_sample, out_sample)
        
        return report
    
    def check_data_quality(self, data):
        """数据质量检查"""
        checks = {
            'min_samples': len(data) >= 1000,
            'no_missing': data.isnull().sum().sum() == 0,
            'time_range': self.check_time_range(data),
            'diversity': self.check_diversity(data),
        }
        
        if not all(checks.values()):
            raise ValueError(f"数据质量检查失败: {checks}")
    
    def generate_report(self, in_sample, out_sample):
        """生成验证报告"""
        report = {
            'market': self.market_type,
            'timeframe': self.timeframe,
            'in_sample': in_sample,
            'out_sample': out_sample,
            'overfitting_risk': in_sample['accuracy'] - out_sample['accuracy'],
            'recommendation': self.make_recommendation(in_sample, out_sample),
        }
        
        return report
```

## 六、总结与建议

### 6.1 核心原则

1. **数据质量 > 数据数量**
   - 100个高质量样本优于1000个噪声样本
   - 多样性比规模更重要

2. **场景适配 > 通用配置**
   - 不同市场需要不同特征和参数
   - 不要期望一套配置通吃所有场景

3. **严格验证 > 盲目乐观**
   - 样本外测试必不可少
   - 过拟合风险时刻存在

4. **工程优化 > 算法复杂**
   - 简单有效的方法优先
   - 性能优化是规模化前提

### 6.2 优先级建议

**第一阶段（1-2周）**：
1. 在A股日线上验证当前架构
2. 完善特征工程和权重算法
3. 实施基础性能优化（缓存+FAISS）

**第二阶段（2-4周）**：
1. 扩展到加密货币（BTC/ETH）
2. 验证T+0和高波动场景
3. 调整时间衰减和标签定义

**第三阶段（1-2月）**：
1. 支持美股、港股
2. 分钟线数据适配
3. 完整工程优化

### 6.3 风险提示

⚠️ **常见陷阱**：
- 直接用A股配置跑加密货币（会失败）
- 样本量不足就下结论（需要统计显著性）
- 忽略交易成本（回测好看，实盘亏损）
- 过拟合历史数据（样本外表现差）
- 性能未优化就规模化（延迟无法接受）

✅ **正确做法**：
- 每个新市场从头验证
- 小样本测试 → 大样本验证 → 样本外测试
- 严格处理交易成本和滑点
- 交叉验证 + 参数鲁棒性测试
- 先优化性能，再扩大规模
