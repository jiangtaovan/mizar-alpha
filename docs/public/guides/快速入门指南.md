# Mizarα 快速入门指南

> **Mizarα** (mizar-alpha) — 基于向量相似性检索的量化策略框架
> 
> PyPI包名: `mizar-alpha` | 导入名: `mizar` | CLI命令: `mizar`

---

## 📖 概述

Mizarα 是一个创新的量化交易框架，核心技术是基于**向量相似性检索**的预测方法。与传统机器学习不同，我们不训练预测模型，而是通过检索历史上相似的市场状态来预测未来走势。

### 核心原理

```
当前市场状态 → 特征向量化 → 检索历史相似状态 → 统计后续走势 → 生成预测信号
```

### 主要特性

- ✅ **向量相似性检索**：基于高维技术指标状态的相似性匹配
- ✅ **实时预测**：快速检索Top-K相似历史片段
- ✅ **策略回测**：内置回测引擎验证策略有效性
- ✅ **灵活配置**：支持自定义特征、权重和参数
- ✅ **API服务**：RESTful API支持集成到交易系统

## 📊 核心概念

### 市场状态向量

每个交易日的技术指标状态被编码为一个高维向量：

```python
{
    'date': '2026-04-21',
    'symbol': '000001',
    'features': {
        'MA_SMA_5': 10.5,      # 5日均线
        'MA_SMA_20': 10.2,     # 20日均线
        'RSI_RSI_12': 55.3,    # RSI指标
        'MACD_macd': 0.15,     # MACD
        'ATR_ATR': 2.5,        # 波动率
        # ... 30-50个技术指标
    }
}
```

### 相似性检索

通过计算向量距离（如欧氏距离、余弦相似度），找到历史上最相似的市场状态：

```python
# 查询当前状态的相似历史
similar_states = db.query(
    features=current_features,
    top_k=10,              # 返回最相似的10个
    weighting_method='distance'  # 距离加权
)
```

### 预测生成

基于相似状态的后续表现，统计预测信号：

```python
{
    'prediction': {
        'avg_ret_1d': 0.8,           # 平均1日收益率
        'up_probability': 0.7,       # 上涨概率70%
        'label_distribution': {      # 标签分布
            '大涨': 2,
            '小涨': 5,
            '小跌': 2,
            '大跌': 1
        }
    }
}
```

## 🔧 使用方式

### 方式一：命令行工具

```bash
# 构建向量数据库
mizar build-db --config config/system_config.yaml

# 执行预测
mizar predict --stock 000001 --top-k 10

# 运行回测
mizar backtest --config config/backtest_config.yaml

# 更新数据
mizar update --date 2026-04-21
```

### 方式二：Python API

```python
from mizar import PredictionService

# 初始化服务
service = PredictionService()

# 执行预测
result = service.predict(
    symbol='000001',
    date='2026-04-21',
    top_k=10
)

print(f"上涨概率: {result.up_probability:.2%}")
print(f"预期收益: {result.avg_return:.2f}%")
```

### 方式三：RESTful API

```bash
# 启动API服务
mizar serve --port 8000

# 查询预测
curl -X POST "http://localhost:8000/query" \
  -H "Content-Type: application/json" \
  -d '{
    "features": {
      "MA_SMA_5": 10.5,
      "MA_SMA_20": 10.2,
      "RSI_RSI_12": 55.3
    },
    "top_k": 10
  }'
```

## ⚙️ 配置说明

### 系统配置 (system_config.yaml)

```yaml
data:
  source_type: csv              # 数据源类型
  data_path: ./data/raw/*.csv   # 数据路径
  date_column: date             # 日期列名
  symbol_column: symbol         # 股票代码列名
  close_column: current_price   # 收盘价列名

vector_db:
  persist_directory: ./data/chroma_db  # 向量库路径
  n_components: 0.95                   # PCA保留方差比例

api:
  port: 8000                    # API端口
  host: 0.0.0.0                # 监听地址
```

### 特征配置 (feature_config.yaml)

```yaml
features:
  - MA_SMA_5        # 简单移动平均
  - MA_SMA_20
  - RSI_RSI_12      # 相对强弱指标
  - MACD_macd       # MACD指标
  - ATR_ATR         # 平均真实波幅
  # ... 建议30-50个特征

normalization:
  method: minmax    # 归一化方法: minmax/zscore
  
pca:
  n_components: 0.95  # 保留95%方差
```

## 📈 典型工作流

### 1. 准备数据

将技术指标数据放入 `data/raw/` 目录：

```csv
date,symbol,current_price,MA_SMA_5,MA_SMA_20,RSI_RSI_12,MACD_macd
2026-04-01,000001,10.5,10.2,10.0,55.3,0.15
2026-04-02,000001,10.6,10.3,10.1,56.8,0.18
```

### 2. 构建向量数据库

```bash
mizar build-db
```

执行流程：
1. 加载历史数据
2. 计算未来收益标签
3. 特征工程（筛选、归一化、PCA）
4. 构建 ChromaDB 向量库

### 3. 执行预测

```bash
mizar predict --stock 000001
```

### 4. 回测验证

```bash
mizar backtest --start-date 2025-01-01 --end-date 2026-04-21
```

### 5. 部署服务

```bash
mizar serve
```

## ❓ 常见问题

### Q: 需要多少历史数据？

A: 建议至少2-3年的日线数据，约500-750个交易日。数据越多，检索效果越好。

### Q: 支持哪些市场？

A: 理论上支持任何有连续交易数据的市场：A股、港股、美股、加密货币等。

### Q: 预测准确率如何？

A: 根据历史回测，在特定市场条件下可达55-65%的上涨概率预测准确率。但请注意，历史表现不代表未来收益。

### Q: 如何选择合适的特征？

A: 建议使用30-50个技术指标，涵盖趋势、动量、波动率、成交量等维度。可参考 `feature_config.yaml` 的默认配置。

### Q: 可以实时预测吗？

A: 可以。单次查询响应时间约200ms，支持实时预测场景。
