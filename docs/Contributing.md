## 如何报告问题

如果您发现了 bug、有改进建议或想讨论新功能，请通过 [GitHub Issues](链接) 提交。在提交前，建议先搜索现有 issue 是否已被提及。

一个好的 issue 应该包含：
- **标题**：简洁明了地描述问题。
- **环境**：操作系统、Python 版本、相关依赖版本。
- **复现步骤**：详细说明如何触发该问题（包括使用的数据、命令等）。
- **预期行为**：您期望发生什么。
- **实际行为**：实际发生了什么（最好附上日志或截图）。
- **可能的解决方案**（可选）：如果您对问题有初步分析，欢迎提出。

对于 bug 报告，请尽量提供最小化可复现代码或数据样例（注意脱敏）。

## 如何贡献代码

我们使用标准的 GitHub 协作流程：

### 分支管理
- 主分支为 `main`，始终保持稳定可运行状态（但可能包含未发布的新特性）。
- 开发新功能或修复 bug 时，请从 `main` 创建新分支，命名规范：
  - `feature/short-description`（新功能）
  - `fix/short-description`（bug 修复）
  - `docs/short-description`（文档更新）
  - `test/short-description`（测试相关）
- 分支应尽量专注于单一目的，避免包含多个不相关的改动。

### 提交 Pull Request
1. 确保您的分支已基于最新的 `main` 并已解决冲突。
2. 在提交 PR 前，运行现有测试确保没有破坏已有功能（如果测试不完善，请尽量手动验证核心流程）。
3. 在 PR 描述中清晰说明：
   - 做了什么改动
   - 为什么需要这个改动
   - 如何验证改动有效（附上测试结果、回测截图等）
4. 如果改动涉及用户配置或 API 变化，请更新相关文档。
5. PR 至少需要一名维护者审核通过后方可合并。

### 代码规范
- **Python 版本**：项目使用 Python 3.10+，请确保代码兼容。
- **风格**：遵循 [PEP 8](https://www.python.org/dev/peps/pep-0008/)。建议使用 `black` 和 `isort` 自动格式化。
- **类型注解**：鼓励为新代码添加类型注解（`typing` 模块）。
- **文档字符串**：使用 Google 风格或 NumPy 风格的 docstring 为公共函数/类添加说明。
- **日志**：使用 `loguru` 进行日志记录，避免直接使用 `print`。
- **配置**：新增配置项请添加到 `config/system_config.yaml` 或对应的配置文件中，并提供默认值。

### 测试要求
目前项目测试覆盖不足，但我们鼓励贡献者补充测试。新增功能或修复 bug 时，请尽量添加相应的单元测试或集成测试（位于 `tests/` 目录）。对于核心模块（如特征计算、相似性检索、预测统计），更应提供测试用例。

运行测试（如已配置）：
```bash
pytest tests/
```

### 提交信息格式
请使用清晰、有意义的提交信息，建议采用以下格式：
```
<type>(<scope>): <subject>

<body>

<footer>
```
- `type`：`feat`（新功能）、`fix`（修复）、`docs`（文档）、`style`（代码格式）、`refactor`（重构）、`test`（测试）、`chore`（构建/工具变动）
- `scope`：受影响的模块（如 `data`、`features`、`retrieval`、`api`）
- `subject`：简短描述（不超过 50 字符）
- `body`：详细说明（可选）
- `footer`：关闭 issue 等信息（如 `Closes #123`）

示例：
```
feat(retrieval): add adaptive K selection based on distance distribution

- Automatically adjusts K when similar samples are scarce.
- Adds a new config option `adaptive_k`.
- Closes #45
```

## 贡献范围

我们特别欢迎以下方向的贡献（优先级由高到低）：

### 🔧 核心缺陷修复
- **回测统计逻辑修正**：修复盈利/亏损判断反转的错误，确保平均盈利/亏损、盈亏比等指标正确。
- **数据复权支持**：在数据加载时自动进行后复权处理，并提供选项。
- **严格样本外验证框架**：实现滚动窗口回测，自动分离训练与测试集，输出稳定性报告。

### 🚀 功能增强
- **更多特征**：添加新的技术指标或自定义特征，扩展 `features/indicator.py`。
- **更多向量数据库**：实现 FAISS、Milvus、Qdrant 等适配器，通过配置一键切换。
- **风险控制模块**：添加止损、止盈、基于波动率的仓位管理等功能。
- **多周期支持**：支持 3 日、10 日等更多预测周期，并允许同时输出多周期信号。
- **多资产组合**：实现组合级别信号生成与风险预算。

### 📈 自动化与优化工具
- **参数优化器**：提供网格搜索、随机搜索、贝叶斯优化脚本，自动寻找最优 `top_k`、阈值等参数。
- **特征选择**：基于重要性或相关性自动筛选特征，降低维度。

### 🧪 测试与质量保障
- **单元测试**：为核心模块编写测试，提高覆盖率。
- **集成测试**：端到端测试完整流程。
- **代码规范**：统一代码风格，添加类型注解。

### 📚 文档与示例
- **更多回测案例**：贡献不同市场（A 股、美股、加密货币）的回测结果与分析。
- **参数调优指南**：分享针对特定市场找到的最佳参数组合。
- **可视化工具**：开发仪表板查看特征分布、相似状态、回测曲线。
